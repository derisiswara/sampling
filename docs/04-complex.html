Multi-stage sampling, Design Effect, dan metode estimasi varians

Survei nasional jarang menggunakan SRS murni — mereka menerapkan stratified multi-stage cluster sampling yang mengkombinasikan beberapa teknik. Bab ini membahas bagaimana mendesain dan menganalisis survei kompleks tersebut (Heeringa dkk., 2017; Lumley, 2010).

1 Multi-Stage Sampling

1.1 Two-Stage Sampling

  1. Stage 1 (PSU — Primary Sampling Unit): Pilih cluster dari populasi. Contoh: kecamatan.
  2. Stage 2 (SSU — Secondary Sampling Unit): Dari PSU terpilih, pilih subsample. Contoh: rumah tangga.

1.2 Contoh: Desain Mirip SUSENAS

BPS SUSENAS menggunakan desain stratified two-stage:

  • Stratum: provinsi × urban/rural
  • PSU: blok sensus (dipilih PPS berdasarkan jumlah rumah tangga)
  • SSU: 10 rumah tangga per blok sensus (dipilih systematic)

1.3 Implementasi Two-Stage Design

  • R
  • Python
Kode 
set.seed(2024)

# Stage 1: Select PSUs (districts) with PPS
district_info <- pop |>
  summarise(
    N_i = n(),
    mean_income = mean(income),
    .by = c(province, district, urban_rural)
  ) |>
  mutate(prob_pps = N_i / sum(N_i))

n_psu <- 15
m_ssu <- 20  # SSU per PSU

# PPS systematic selection
cum_prob <- cumsum(district_info$prob_pps)
interval <- 1 / n_psu
start <- runif(1, 0, interval)
select_pts <- start + (0:(n_psu - 1)) * interval

psu_selected <- sapply(select_pts, function(p) {
  district_info$district[min(which(cum_prob >= p))]
})

# Stage 2: SRS within each PSU
two_stage_sample <- pop |>
  filter(district %in% psu_selected) |>
  slice_sample(n = m_ssu, by = district)

# Setup survey design object
two_stage_sample <- two_stage_sample |>
  left_join(
    district_info |> select(district, N_i, prob_pps),
    by = "district"
  ) |>
  mutate(
    psu_prob = n_psu * prob_pps,
    ssu_prob = m_ssu / N_i,
    final_weight = 1 / (psu_prob * ssu_prob)
  )

# Survey design
des <- svydesign(
  id = ~district,
  strata = NULL,
  weights = ~final_weight,
  data = two_stage_sample,
  fpc = NULL
)

# Estimates
est_mean <- svymean(~income, des)
est_total <- svytotal(~income, des)

tibble(
  Parameter = c("Mean Income", "Total Income"),
  Estimate = c(

Two-stage sampling: 15 district (PSU) dipilih PPS, 20 RT (SSU) per district.

Kode 
import numpy as np
import pandas as pd

np.random.seed(2024)
pop_py = r.pop.copy()

# Stage 1: Select districts
dist_info = pop_py.groupby('district').agg(
    N_i=('income', 'size'),
    mean_income=('income', 'mean')
).reset_index()
dist_info['prob'] = dist_info['N_i'] / dist_info['N_i'].sum()

n_psu = 15
m_ssu = 20

cum_prob = dist_info['prob'].cumsum().values
interval = 1 / n_psu
start = np.random.uniform(0, interval)
pts = start + np.arange(n_psu) * interval
psu_sel = [dist_info.iloc[np.searchsorted(cum_prob, p)]['district']
           for p in pts]

# Stage 2: SRS within PSU
samples = []
for d in psu_sel:
    d_data = pop_py[pop_py['district'] == d]
    s = d_data.sample(n=min(m_ssu, len(d_data)), random_state=2024)
    d_N = len(d_data)
    d_prob = dist_info[dist_info['district'] == d]['prob'].values[0]
    s = s.copy()
    s['weight'] = 1 / (n_psu * d_prob * m_ssu / d_N)
    samples.append(s)

sample_2s = pd.concat(samples)

# Weighted mean
weighted_mean = np.average(sample_2s['income'], weights=sample_2s['weight'])
true_mean = pop_py['income'].mean()

results = pd.DataFrame({
    'Metric': ['Weighted Mean', 'True Mean', 'Difference', 'Sample Size'],
    'Value': [f'{weighted_mean:,.0f}', f'{true_mean:,.0f}',
              f'{weighted_mean - true_mean:,.0f}', f'{len(sample_2s)}']
})

Desain survei nasional tipikal menggabungkan stratifikasi dengan multi-stage clustering:

Kode 
set.seed(2024)

# Create strata
pop <- pop |>
  mutate(stratum = paste(region, urban_rural, sep = "_"))

strata_list <- unique(pop$stratum)
n_psu_per_stratum <- 2
m_per_psu <- 15

# Stage 1: Select PSUs within each stratum
complex_sample_list <- list()
for (st in strata_list) {
  st_pop <- pop |> filter(stratum == st)
  st_districts <- unique(st_pop$district)

  if (length(st_districts) >= n_psu_per_stratum) {
    sel_psu <- sample(st_districts, n_psu_per_stratum)
  } else {
    sel_psu <- st_districts
  }

  # Stage 2: SRS within PSU
  st_sample <- st_pop |>
    filter(district %in% sel_psu) |>
    group_by(district) |>
    group_modify(~ slice_sample(.x, n = min(m_per_psu, nrow(.x)))) |>
    ungroup() |>
    mutate(
      stratum_id = st,
      psu_id = district,
      n_psu_in_stratum = length(sel_psu)
    )

  complex_sample_list[[st]] <- st_sample
}

complex_sample <- bind_rows(complex_sample_list)

# Compute weights (simplified)
complex_sample <- complex_sample |>
  left_join(
    pop |> summarise(N_h = n(), .by = stratum),
    by = c("stratum_id" = "stratum")
  ) |>
  mutate(
    base_weight = N_h / n(),
    .by = stratum_id
  )

# Survey design
des_complex <- svydesign(
  id = ~psu_id,
  strata = ~stratum_id,
  weights = ~base_weight,
  data = complex_sample,
  nest = TRUE
)

3 Design Effect (DEFF) & Effective Sample Size

3.1 DEFF karena Clustering

\[ \text{DEFF}_{cl} = 1 + (\bar{m} - 1)\rho \tag{1}\]

di mana \(\bar{m}\) = rata-rata unit per cluster, \(\rho\) = ICC.

3.2 DEFF karena Weighting

\[ \text{DEFF}_w = 1 + \text{CV}(w)^2 \tag{2}\]

di mana \(\text{CV}(w) = \text{SD}(w) / \text{mean}(w)\) adalah coefficient of variation dari weights.

3.3 Combined DEFF

\[ \text{DEFF} \approx \text{DEFF}_{cl} \times \text{DEFF}_w \tag{3}\]

3.4 Effective Sample Size

\[ n_{eff} = \frac{n}{\text{DEFF}} \tag{4}\]

PeringatanInterpretasi Praktis

Jika \(n = 1000\) dan \(\text{DEFF} = 2.5\), maka \(n_{eff} = 400\). Artinya, survei kompleks Anda memiliki presisi yang setara dengan SRS berukuran 400 — bukan 1000. Ini memiliki implikasi besar terhadap sample size planning.

4 Variance Estimation untuk Complex Designs

Untuk desain kompleks, variance tidak bisa dihitung dengan formula SRS sederhana. Empat metode utama:

4.1 Taylor Series Linearization (TSL)

Metode default di survey package. Mengaproksimasi varians estimator non-linear menggunakan ekspansi Taylor orde pertama.

\[ \hat{V}_{TSL}(\hat{\theta}) \approx \sum_{h=1}^H \frac{n_h}{n_h - 1} \sum_{i=1}^{n_h} (e_{hi} - \bar{e}_h)^2 \tag{5}\]

di mana \(e_{hi}\) = linearized residual dari PSU ke-\(i\) di stratum \(h\).

4.2 Replication Methods

Metode Prinsip Jumlah Replikasi Cocok untuk
BRR Half-sample replicates \(n_h = 2\) per stratum required Desain paired PSU
Jackknife (JRR) Drop-one PSU \(\sum n_h\) replikasi General, robust
Bootstrap Resample PSUs with replacement B (biasanya 200-500) Paling fleksibel

set.seed(2024)

# Design with paired PSUs for BRR compatibility
# Simplified: use 2 PSUs per stratum
strata_v <- unique(pop$region)
n_psu_v <- 2  # for BRR compatibility
m_v <- 30

var_sample_list <- list()
for (st in strata_v) {
  st_pop <- pop |> filter(region == st)
  dists <- unique(st_pop$district)
  sel <- sample(dists, min(n_psu_v, length(dists)))
  ss <- st_pop |>
    filter(district %in% sel) |>
    group_by(district) |>
    group_modify(~ slice_sample(.x, n = min(m_v, nrow(.x)))) |>
    ungroup() |>
    mutate(stratum_v = st, psu_v = district)
  var_sample_list[[st]] <- ss
}

var_sample <- bind_rows(var_sample_list) |>
  mutate(wt = N / n())

# TSL design
des_tsl <- svydesign(id = ~psu_v, strata = ~stratum_v,
                     weights = ~wt, data = var_sample, nest = TRUE)

# JRR design
des_jrr <- as.svrepdesign(des_tsl, type = "JKn")

# Bootstrap design
des_boot <- as.svrepdesign(des_tsl, type = "bootstrap", replicates = 200)

import numpy as np
import pandas as pd

# Simplified comparison using manual jackknife
sample_py = r.var_sample.copy()
clusters = sample_py['psu_v'].unique()
income_vals = sample_py['income'].values
weights = sample_py['wt'].values

# Weighted mean
theta_hat = np.average(income_vals, weights=weights)

# Jackknife: drop one cluster at a time
jack_estimates = []
for c in clusters:
    mask = sample_py['psu_v'] != c
    jack_est = np.average(income_vals[mask], weights=weights[mask])
    jack_estimates.append(jack_est)

jack_estimates = np.array(jack_estimates)
n_c = len(clusters)
jack_var = ((n_c - 1) / n_c) * np.sum((jack_estimates - theta_hat)**2)
jack_se = np.sqrt(jack_var)

results = pd.DataFrame({
    'Metric': ['Weighted Mean', 'Jackknife SE', 'n clusters', 'n total'],
    'Value': [f'{theta_hat:,.0f}', f'{jack_se:,.0f}',

})

PeringatanCommon Pitfall: Subpopulation Analysis

SALAH: Subset data dulu, baru buat survey design.

# ❌ SALAH — mengabaikan design structure
urban_data <- var_sample |> filter(urban_rural == "Urban")
des_wrong <- svydesign(id = ~psu_v, strata = ~stratum_v,
                       weights = ~wt, data = urban_data)

BENAR: Gunakan subset() pada survey design object, atau svyby().

# ✅ BENAR — preserves design information
des_urban <- subset(des, urban_rural == "Urban")
svymean(~income, des_urban)

Memfilter sebelum svydesign() menghapus informasi tentang PSU dan strata yang tidak terpilih, menghasilkan standard error yang underestimated.

5.6 Key Takeaways

  1. Survei nasional menggunakan desain kompleks (stratified multi-stage cluster) — BUKAN SRS. Analisis harus memperhitungkan design features.
  2. DEFF menghubungkan presisi aktual dengan presisi SRS. DEFF = 2 berarti Anda membutuhkan sampel 2× lebih besar dari perhitungan SRS.
  3. Selalu gunakan svydesign() (atau as_survey_design()) di R. Jangan pernah ignore weights, PSU, atau strata.
  4. Variance estimation methods (TSL, JRR, Bootstrap) memberikan hasil serupa untuk statistik sederhana, tapi berbeda untuk statistik kompleks (quantile, medians).
  5. Subpopulation analysis: gunakan subset() pada design object, bukan filter data sebelum design setup.

5.7 Exercises

## Multi-Stage Sampling

### Two-Stage Sampling

1. **Stage 1 (PSU — Primary Sampling Unit):** Pilih cluster dari populasi. Contoh: kecamatan.
2. **Stage 2 (SSU — Secondary Sampling Unit):** Dari PSU terpilih, pilih subsample. Contoh: rumah tangga.

### Contoh: Desain Mirip SUSENAS

BPS SUSENAS menggunakan desain stratified two-stage:

- **Stratum:** provinsi × urban/rural
- **PSU:** blok sensus (dipilih PPS berdasarkan jumlah rumah tangga)
- **SSU:** 10 rumah tangga per blok sensus (dipilih systematic)

### Implementasi Two-Stage Design

::: {.panel-tabset}

#### R

```{r}
#| label: two-stage-design
#| fig-cap: "Two-stage sampling: 15 district (PSU) dipilih PPS, 20 RT (SSU) per district."
#| fig-width: 9
#| fig-height: 5

set.seed(2024)

# Stage 1: Select PSUs (districts) with PPS
district_info <- pop |>
  summarise(
    N_i = n(),
    mean_income = mean(income),
    .by = c(province, district, urban_rural)
  ) |>
  mutate(prob_pps = N_i / sum(N_i))

n_psu <- 15
m_ssu <- 20  # SSU per PSU

# PPS systematic selection
cum_prob <- cumsum(district_info$prob_pps)
interval <- 1 / n_psu
start <- runif(1, 0, interval)
select_pts <- start + (0:(n_psu - 1)) * interval

psu_selected <- sapply(select_pts, function(p) {
  district_info$district[min(which(cum_prob >= p))]
})

# Stage 2: SRS within each PSU
two_stage_sample <- pop |>
  filter(district %in% psu_selected) |>
  slice_sample(n = m_ssu, by = district)

# Setup survey design object
two_stage_sample <- two_stage_sample |>
  left_join(
    district_info |> select(district, N_i, prob_pps),
    by = "district"
  ) |>
  mutate(
    psu_prob = n_psu * prob_pps,
    ssu_prob = m_ssu / N_i,
    final_weight = 1 / (psu_prob * ssu_prob)
  )

# Survey design
des <- svydesign(
  id = ~district,
  strata = NULL,
  weights = ~final_weight,
  data = two_stage_sample,
  fpc = NULL
)

# Estimates
est_mean <- svymean(~income, des)
est_total <- svytotal(~income, des)

tibble(
  Parameter = c("Mean Income", "Total Income"),
  Estimate = c(
    format(round(coef(est_mean)), big.mark = ","),
    format(round(coef(est_total)), big.mark = ",")
  ),
  SE = c(
    format(round(SE(est_mean)), big.mark = ","),
    format(round(SE(est_total)), big.mark = ",")
  ),
  `CI Lower` = c(
    format(round(confint(est_mean)[1]), big.mark = ","),
    format(round(confint(est_total)[1]), big.mark = ",")
  ),
  `CI Upper` = c(
    format(round(confint(est_mean)[2]), big.mark = ","),
    format(round(confint(est_total)[2]), big.mark = ",")
  )
) |>
  gt() |>
  tab_header(title = "Two-Stage Design Estimates (survey package)")
```

```{python}
#| label: two-stage-design-py
#| fig-cap: "Two-stage sampling estimates (Python)."

import numpy as np
import pandas as pd

np.random.seed(2024)
pop_py = r.pop.copy()

# Stage 1: Select districts
dist_info = pop_py.groupby('district').agg(
    N_i=('income', 'size'),
    mean_income=('income', 'mean')
).reset_index()
dist_info['prob'] = dist_info['N_i'] / dist_info['N_i'].sum()

n_psu = 15
m_ssu = 20

cum_prob = dist_info['prob'].cumsum().values
interval = 1 / n_psu
start = np.random.uniform(0, interval)
pts = start + np.arange(n_psu) * interval
psu_sel = [dist_info.iloc[np.searchsorted(cum_prob, p)]['district']
           for p in pts]

# Stage 2: SRS within PSU
samples = []
for d in psu_sel:
    d_data = pop_py[pop_py['district'] == d]
    s = d_data.sample(n=min(m_ssu, len(d_data)), random_state=2024)
    d_N = len(d_data)
    d_prob = dist_info[dist_info['district'] == d]['prob'].values[0]
    s = s.copy()
    s['weight'] = 1 / (n_psu * d_prob * m_ssu / d_N)
    samples.append(s)

sample_2s = pd.concat(samples)

# Weighted mean
weighted_mean = np.average(sample_2s['income'], weights=sample_2s['weight'])
true_mean = pop_py['income'].mean()

results = pd.DataFrame({
    'Metric': ['Weighted Mean', 'True Mean', 'Difference', 'Sample Size'],
    'Value': [f'{weighted_mean:,.0f}', f'{true_mean:,.0f}',
              f'{weighted_mean - true_mean:,.0f}', f'{len(sample_2s)}']
})
results
```

```{r}
#| label: complex-design
#| fig-cap: "Stratified two-stage cluster design menggunakan region × urban_rural sebagai strata."

set.seed(2024)

# Create strata
pop <- pop |>
  mutate(stratum = paste(region, urban_rural, sep = "_"))

strata_list <- unique(pop$stratum)
n_psu_per_stratum <- 2
m_per_psu <- 15

# Stage 1: Select PSUs within each stratum
complex_sample_list <- list()
for (st in strata_list) {
  st_pop <- pop |> filter(stratum == st)
  st_districts <- unique(st_pop$district)

  if (length(st_districts) >= n_psu_per_stratum) {
    sel_psu <- sample(st_districts, n_psu_per_stratum)
  } else {
    sel_psu <- st_districts
  }

  # Stage 2: SRS within PSU
  st_sample <- st_pop |>
    filter(district %in% sel_psu) |>
    group_by(district) |>
    group_modify(~ slice_sample(.x, n = min(m_per_psu, nrow(.x)))) |>
    ungroup() |>
    mutate(
      stratum_id = st,
      psu_id = district,
      n_psu_in_stratum = length(sel_psu)
    )

  complex_sample_list[[st]] <- st_sample
}

complex_sample <- bind_rows(complex_sample_list)

# Compute weights (simplified)
complex_sample <- complex_sample |>
  left_join(
    pop |> summarise(N_h = n(), .by = stratum),
    by = c("stratum_id" = "stratum")
  ) |>
  mutate(
    base_weight = N_h / n(),
    .by = stratum_id
  )

# Survey design
des_complex <- svydesign(
  id = ~psu_id,
  strata = ~stratum_id,
  weights = ~base_weight,
  data = complex_sample,
  nest = TRUE
)

# Estimates
est_complex <- svymean(~income, des_complex)
est_by_region <- svyby(~income, ~region, des_complex, svymean)

est_by_region |>
  as_tibble() |>
```

## Design Effect (DEFF) & Effective Sample Size

### DEFF karena Clustering

$$
\text{DEFF}_{cl} = 1 + (\bar{m} - 1)\rho
$$ {#eq-deff-cl}

di mana $\bar{m}$ = rata-rata unit per cluster, $\rho$ = ICC.

### DEFF karena Weighting

$$
\text{DEFF}_w = 1 + \text{CV}(w)^2
$$ {#eq-deff-w}

di mana $\text{CV}(w) = \text{SD}(w) / \text{mean}(w)$ adalah coefficient of variation dari weights.

### Combined DEFF

$$
\text{DEFF} \approx \text{DEFF}_{cl} \times \text{DEFF}_w
$$ {#eq-deff-combined}

### Effective Sample Size

$$
n_{eff} = \frac{n}{\text{DEFF}}
$$ {#eq-neff}

::: {.callout-warning}
## Interpretasi Praktis
Jika $n = 1000$ dan $\text{DEFF} = 2.5$, maka $n_{eff} = 400$. Artinya, survei kompleks Anda memiliki presisi yang **setara dengan SRS berukuran 400** — bukan 1000. Ini memiliki implikasi besar terhadap sample size planning.
:::

## Variance Estimation untuk Complex Designs

Untuk desain kompleks, variance **tidak** bisa dihitung dengan formula SRS sederhana. Empat metode utama:

### Taylor Series Linearization (TSL)

Metode default di `survey` package. Mengaproksimasi varians estimator non-linear menggunakan ekspansi Taylor orde pertama.

$$
\hat{V}_{TSL}(\hat{\theta}) \approx \sum_{h=1}^H \frac{n_h}{n_h - 1} \sum_{i=1}^{n_h} (e_{hi} - \bar{e}_h)^2
$$ {#eq-tsl}

di mana $e_{hi}$ = linearized residual dari PSU ke-$i$ di stratum $h$.

### Replication Methods

| Metode | Prinsip | Jumlah Replikasi | Cocok untuk |
|--------|---------|-----------------|-------------|
| **BRR** | Half-sample replicates | $n_h = 2$ per stratum required | Desain paired PSU |
| **Jackknife (JRR)** | Drop-one PSU | $\sum n_h$ replikasi | General, robust |
| **Bootstrap** | Resample PSUs with replacement | B (biasanya 200-500) | Paling fleksibel |

### Implementasi Variance Estimation

::: {.panel-tabset}

#### R

```{r}
#| label: variance-estimation
#| tbl-cap: "Perbandingan metode estimasi varians untuk desain survei kompleks."

set.seed(2024)

# Design with paired PSUs for BRR compatibility
# Simplified: use 2 PSUs per stratum
strata_v <- unique(pop$region)
n_psu_v <- 2  # for BRR compatibility
m_v <- 30

var_sample_list <- list()
for (st in strata_v) {
  st_pop <- pop |> filter(region == st)
  dists <- unique(st_pop$district)
  sel <- sample(dists, min(n_psu_v, length(dists)))
  ss <- st_pop |>
    filter(district %in% sel) |>
    group_by(district) |>
    group_modify(~ slice_sample(.x, n = min(m_v, nrow(.x)))) |>
    ungroup() |>
    mutate(stratum_v = st, psu_v = district)
  var_sample_list[[st]] <- ss
}

var_sample <- bind_rows(var_sample_list) |>
  mutate(wt = N / n())

# TSL design
des_tsl <- svydesign(id = ~psu_v, strata = ~stratum_v,
                     weights = ~wt, data = var_sample, nest = TRUE)

# JRR design
des_jrr <- as.svrepdesign(des_tsl, type = "JKn")

# Bootstrap design
des_boot <- as.svrepdesign(des_tsl, type = "bootstrap", replicates = 200)

# Estimates comparison
tsl_est <- svymean(~income, des_tsl)
jrr_est <- svymean(~income, des_jrr)
boot_est <- svymean(~income, des_boot)

tibble(
  Method = c("TSL (Taylor)", "Jackknife (JKn)", "Bootstrap (B=200)"),
  Estimate = c(coef(tsl_est), coef(jrr_est), coef(boot_est)),
  SE = c(SE(tsl_est), SE(jrr_est), SE(boot_est))
) |>
  mutate(
    Estimate = format(round(Estimate), big.mark = ","),
    SE = format(round(SE), big.mark = ",")
  ) |>
  gt() |>
  tab_header(
    title = "Variance Estimation Methods Comparison",
    subtitle = "Estimasi mean income — semua metode menghasilkan SE yang serupa"
```

```{python}
#| label: variance-estimation-py

import numpy as np
import pandas as pd

# Simplified comparison using manual jackknife
sample_py = r.var_sample.copy()
clusters = sample_py['psu_v'].unique()
income_vals = sample_py['income'].values
weights = sample_py['wt'].values

# Weighted mean
theta_hat = np.average(income_vals, weights=weights)

# Jackknife: drop one cluster at a time
jack_estimates = []
for c in clusters:
    mask = sample_py['psu_v'] != c
    jack_est = np.average(income_vals[mask], weights=weights[mask])
    jack_estimates.append(jack_est)

jack_estimates = np.array(jack_estimates)
n_c = len(clusters)
jack_var = ((n_c - 1) / n_c) * np.sum((jack_estimates - theta_hat)**2)
jack_se = np.sqrt(jack_var)

results = pd.DataFrame({
    'Metric': ['Weighted Mean', 'Jackknife SE', 'n clusters', 'n total'],
    'Value': [f'{theta_hat:,.0f}', f'{jack_se:,.0f}',
              str(n_c), str(len(income_vals))]
})
```

## Common Pitfall: Subpopulation Analysis

**SALAH:** Subset data dulu, baru buat survey design.

```r
# ❌ SALAH — mengabaikan design structure
urban_data <- var_sample |> filter(urban_rural == "Urban")
des_wrong <- svydesign(id = ~psu_v, strata = ~stratum_v,
                       weights = ~wt, data = urban_data)
```

**BENAR:** Gunakan `subset()` pada survey design object, atau `svyby()`.

```r
# ✅ BENAR — preserves design information
des_urban <- subset(des, urban_rural == "Urban")
svymean(~income, des_urban)
```

Memfilter sebelum `svydesign()` menghapus informasi tentang PSU dan strata yang tidak terpilih, menghasilkan standard error yang **underestimated**.

::: {.takeaways}
### Key Takeaways

1. **Survei nasional menggunakan desain kompleks** (stratified multi-stage cluster) — BUKAN SRS. Analisis harus memperhitungkan design features.
2. **DEFF menghubungkan presisi aktual dengan presisi SRS.** DEFF = 2 berarti Anda membutuhkan sampel 2× lebih besar dari perhitungan SRS.
3. **Selalu gunakan `svydesign()`** (atau `as_survey_design()`) di R. Jangan pernah ignore weights, PSU, atau strata.
4. **Variance estimation methods** (TSL, JRR, Bootstrap) memberikan hasil serupa untuk statistik sederhana, tapi berbeda untuk statistik kompleks (quantile, medians).
5. **Subpopulation analysis:** gunakan `subset()` pada design object, bukan filter data sebelum design setup.
