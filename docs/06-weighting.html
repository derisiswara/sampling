Survey Weighting & Adjustment

  • Tampilkan Semua Kode
  • Sembunyikan Semua Kode
  • Lihat Sumber

Base weight, non-response adjustment, calibration, dan diagnostik

Survey weights menjembatani gap antara sampel dan populasi. Tanpa weighting yang benar, estimasi dari desain survei kompleks akan biased (Kish, 1992; Valliant dkk., 2018).

1 Base Weight

1.1 Definisi

Base weight = inverse of selection probability:

\[ w_i = \frac{1}{\pi_i} \tag{1}\]

di mana \(\pi_i\) = probabilitas inklusi unit \(i\).

Untuk desain two-stage (PPS stage 1, SRS stage 2):

\[ w_i = \frac{1}{\pi_{i1} \times \pi_{i2|1}} = \frac{1}{n_1 \cdot p_i} \times \frac{N_{c(i)}}{n_{c(i)}} \]

di mana \(p_i\) = probability cluster terpilih, \(N_{c(i)}\) = jumlah unit dalam cluster, \(n_{c(i)}\) = unit yang disampling.

CatatanSelf-Weighting Design

Jika semua unit memiliki base weight sama (\(w_i = w\) untuk semua \(i\)), desain disebut self-weighting. Ini terjadi ketika PPS digunakan di stage 1 dan equal-size SRS di stage 2. Self-weighting designs menghasilkan \(\text{DEFF}_w = 1\).

set.seed(2024)

# Simulate two-stage design
dist_info <- pop |>
  summarise(N_d = n(), .by = district) |>
  mutate(prob_d = N_d / sum(N_d))

n_psu <- 15
m_ssu <- 20

# PPS selection of districts
cum_p <- cumsum(dist_info$prob_d)
int <- 1 / n_psu
st <- runif(1, 0, int)
pts <- st + (0:(n_psu - 1)) * int
sel_districts <- sapply(pts, function(p) {
  dist_info$district[min(which(cum_p >= p))]
})

# SRS within selected districts
weighted_sample <- pop |>
  filter(district %in% sel_districts) |>
  slice_sample(n = m_ssu, by = district) |>
  left_join(dist_info, by = "district") |>
  mutate(
    pi_1 = pmin(n_psu * prob_d, 0.999),
    pi_2 = m_ssu / N_d,
    pi_total = pi_1 * pi_2,
    base_weight = 1 / pi_total
  )

# Weight summary
weighted_sample |>
  summarise(
    min_wt = min(base_weight),
    median_wt = median(base_weight),
    mean_wt = mean(base_weight),
    max_wt = max(base_weight),
    cv_wt = sd(base_weight) / mean(base_weight),
    sum_wt = sum(base_weight)
  ) |>
  pivot_longer(everything(), names_to = "Statistic", values_to = "Value") |>
  mutate(Value = format(round(Value, 2), big.mark = ",")) |>
  gt() |>
  tab_header(title = "Base Weight Distribution Summary")

2 Non-Response Adjustment

2.1 Weighting Class Adjustment

Bagi sampel menjadi weighting classes (kelompok homogen terhadap response propensity). Dalam setiap class, inflate weight respondent untuk “mewakili” non-respondent:

\[ w_i^{NR} = w_i \times \frac{n_c}{n_c^{resp}} \tag{2}\]

di mana \(n_c\) = total sampled in class \(c\), \(n_c^{resp}\) = respondents in class \(c\).

2.2 Propensity Model Adjustment

Model probabilitas response menggunakan logistic regression:

\[ \hat{p}_i = P(\text{respond} \mid X_i) = \text{logit}^{-1}(\beta_0 + \beta_1 X_{i1} + \ldots) \tag{3}\]

Non-response adjusted weight: \(w_i^{NR} = w_i / \hat{p}_i\)

set.seed(2024)

# Simulate non-response: higher income & urban less likely to respond
weighted_sample <- weighted_sample |>
  mutate(
    resp_prob = plogis(-0.5 + 0.3 * (urban_rural == "Rural") -
                         0.00000005 * income + 0.05 * household_size),
    responded = rbinom(n(), 1, resp_prob)
  )

response_rate <- mean(weighted_sample$responded)

# Method 1: Weighting class adjustment (by urban/rural × region)
nr_adj <- weighted_sample |>
  mutate(wt_class = paste(urban_rural, region)) |>
  mutate(
    n_class = n(),
    n_resp = sum(responded),
    nr_factor = n_class / n_resp,
    .by = wt_class
  ) |>
  mutate(
    wt_nr = ifelse(responded == 1, base_weight * nr_factor, NA)
  )

respondents <- nr_adj |> filter(responded == 1)

# Compare estimates

est_nr <- weighted.mean(respondents$income, respondents$wt_nr)

3 Calibration / Raking

3.1 Konsep

Calibration menyesuaikan weights agar distribusi marginal sampel match dengan known population totals dari sensus atau administrative data (Deville & Särndal, 1992).

Raking (iterative proportional fitting) menyesuaikan weights secara bergantian pada setiap variabel sampai konvergensi.

3.2 Implementasi

  • R
  • Python
Kode 
# Known population totals
pop_totals_urban <- pop |>
  count(urban_rural) |>
  mutate(Freq = n) |>
  select(urban_rural, Freq)

pop_totals_region <- pop |>
  count(region) |>
  mutate(Freq = n) |>
  select(region, Freq)

# Design with NR-adjusted weights
des_nr <- svydesign(
  id = ~district,
  weights = ~wt_nr,
  data = respondents
)

# Post-stratification on urban/rural
pop_table_urban <- xtabs(~urban_rural, data = pop)
obs_urban <- sort(unique(respondents$urban_rural))
pop_table_urban <- pop_table_urban[names(pop_table_urban) %in% obs_urban]
des_ps <- postStratify(des_nr, ~urban_rural, pop_table_urban, partial = TRUE)

# Raking on urban/rural AND region
pop_table_region <- xtabs(~region, data = pop)
obs_region <- sort(unique(respondents$region))
pop_table_region <- pop_table_region[names(pop_table_region) %in% obs_region]
des_rake <- rake(des_nr,
                  sample.margins = list(~urban_rural, ~region),
                  population.margins = list(pop_table_urban, pop_table_region))

# Compare
est_pre <- coef(svymean(~income, des_nr))
est_ps <- coef(svymean(~income, des_ps))
est_rake <- coef(svymean(~income, des_rake))

sample_py = r.respondents.copy()

# Known population proportions

pop_region_prop = pop_py['region'].value_counts(normalize=True).to_dict()

# Manual single-step calibration (urban/rural)

4 Weight Trimming

4.1 Mengapa Trimming?

Extreme weights meningkatkan varians estimasi. Trimming meng-cap weights pada percentile tertentu (misalnya P95 atau P99) atau pada kelipatan dari median.

\[ w_i^{trim} = \min(w_i, w_{max}) \]

PeringatanTrade-off

Trimming mengurangi varians tapi memperkenalkan bias. Aturan praktis: trim jika \(\text{CV}(w) > 1.0\) atau jika ada weights yang >10× median (Potter, 1990).

4.2 Implementasi

  • R
  • Python
Kode 
wts <- weights(des_rake)

# Trimming at various percentiles
trim_levels <- c(0.95, 0.97, 0.99, 1.00)

trim_results <- map_dfr(trim_levels, function(p) {
  cap <- quantile(wts, p)
  trimmed <- pmin(wts, cap)
  est <- weighted.mean(respondents$income, trimmed)
  tibble(
    percentile = paste0("P", p * 100),
    cap_value = round(cap),
    mean_wt = mean(trimmed),
    cv_wt = sd(trimmed) / mean(trimmed),
    estimate = est,
    bias = est - true_mean
  )
})

5 Effective Sample Size Setelah Weighting

\[ n_{eff} = \frac{(\sum w_i)^2}{\sum w_i^2} = \frac{n}{1 + \text{CV}(w)^2} \tag{4}\]

  • R
  • Python
Kode 
calc_neff <- function(w) {
  sum(w)^2 / sum(w^2)
}

tibble(
  Stage = c("Base weight", "NR adjusted", "Post-raking", "Trimmed (P95)"),
  n_actual = nrow(respondents),
  n_eff = c(
    calc_neff(respondents$base_weight),
    calc_neff(respondents$wt_nr),
    calc_neff(weights(des_rake)),
    calc_neff(trimmed_wts)
  ),
  CV_weights = c(
    sd(respondents$base_weight) / mean(respondents$base_weight),
    sd(respondents$wt_nr) / mean(respondents$wt_nr),
    sd(weights(des_rake)) / mean(weights(des_rake)),
    sd(trimmed_wts) / mean(trimmed_wts)
  ),
  DEFF_w = c(
    1 + (sd(respondents$base_weight) / mean(respondents$base_weight))^2,
    1 + (sd(respondents$wt_nr) / mean(respondents$wt_nr))^2,
    1 + (sd(weights(des_rake)) / mean(weights(des_rake)))^2,
    1 + (sd(trimmed_wts) / mean(trimmed_wts))^2
  )
) |>
  mutate(across(c(n_eff, CV_weights, DEFF_w), ~round(., 2))) |>
  gt() |>
  tab_header(title = "Effective Sample Size by Weighting Stage")

def calc_neff(w):

wts_rake = np.array(r['weights(des_rake)']) if hasattr(r, 'des_rake') else sample_py['wt_nr'].values

nr_w = sample_py['wt_nr'].values

results = pd.DataFrame({
    'Stage': ['Base weight', 'NR adjusted'],

})

6 Weight Diagnostics

6.1 Checklist Diagnostik

  1. Distribusi weight: histogram, box plot — cari extreme values
  2. CV of weights: target CV < 1.0; CV > 1.5 problematic
  3. Sum of weights: harus ≈ \(N\) (population size)
  4. Ratio max/min weight: jika > 20, pertimbangkan trimming
  5. Marginal distributions: setelah calibration, distribusi variabel kunci harus match populasi

6.2 Key Takeaways

  1. Base weight = 1/π_i — ini adalah starting point. Jangan pernah analyze data survei tanpa weight.
  2. Non-response adjustment mengoreksi bias dari unit yang tidak merespons, menggunakan weighting classes atau propensity models.
  3. Calibration/raking menyesuaikan weight agar distribusi match dengan populasi yang diketahui — mengurangi bias dan bisa meningkatkan presisi.
  4. Weight trimming mengurangi varians dari extreme weights, tapi trade-off dengan sedikit bias.
  5. Selalu lakukan diagnostik: cek CV of weights, distribusi, dan marginal conformity setelah setiap tahap adjustment.

6.3 Exercises

  1. Weight construction. Dari desain two-stage (15 district PPS, 20 RT per district), hitung base weight untuk setiap unit. Verifikasi bahwa \(\sum w_i \approx N\).

  2. Non-response simulation. Simulasikan non-response yang berkorelasi dengan income (higher income = lower response). Bandingkan estimasi mean income: (a) tanpa adjustment, (b) dengan weighting class adjustment (urban/rural × education), (c) dengan propensity model. Mana yang paling efektif mengurangi bias?

  3. Calibration exercise. Lakukan raking pada sampel menggunakan distribusi populasi: urban/rural, region, dan education level. Berapa persen pengurangan bias? Berapa n_eff setelah raking?

Survey weights menjembatani gap antara **sampel** dan **populasi**. Tanpa weighting yang benar, estimasi dari desain survei kompleks akan biased [@valliant2018; @kish1992].

## Base Weight

### Definisi

Base weight = inverse of selection probability:

$$
w_i = \frac{1}{\pi_i}
$$ {#eq-base-weight}

di mana $\pi_i$ = probabilitas inklusi unit $i$.

Untuk desain two-stage (PPS stage 1, SRS stage 2):

$$
w_i = \frac{1}{\pi_{i1} \times \pi_{i2|1}} = \frac{1}{n_1 \cdot p_i} \times \frac{N_{c(i)}}{n_{c(i)}}
$$

di mana $p_i$ = probability cluster terpilih, $N_{c(i)}$ = jumlah unit dalam cluster, $n_{c(i)}$ = unit yang disampling.

::: {.callout-note}
## Self-Weighting Design
Jika semua unit memiliki base weight sama ($w_i = w$ untuk semua $i$), desain disebut **self-weighting**. Ini terjadi ketika PPS digunakan di stage 1 dan equal-size SRS di stage 2. Self-weighting designs menghasilkan $\text{DEFF}_w = 1$.
:::

### Implementasi

::: {.panel-tabset}

#### R

```{r}
#| label: base-weight-demo

set.seed(2024)

# Simulate two-stage design
dist_info <- pop |>
  summarise(N_d = n(), .by = district) |>
  mutate(prob_d = N_d / sum(N_d))

n_psu <- 15
m_ssu <- 20

# PPS selection of districts
cum_p <- cumsum(dist_info$prob_d)
int <- 1 / n_psu
st <- runif(1, 0, int)
pts <- st + (0:(n_psu - 1)) * int
sel_districts <- sapply(pts, function(p) {
  dist_info$district[min(which(cum_p >= p))]
})

# SRS within selected districts
weighted_sample <- pop |>
  filter(district %in% sel_districts) |>
  slice_sample(n = m_ssu, by = district) |>
  left_join(dist_info, by = "district") |>
  mutate(
    pi_1 = pmin(n_psu * prob_d, 0.999),
    pi_2 = m_ssu / N_d,
    pi_total = pi_1 * pi_2,
    base_weight = 1 / pi_total
  )

# Weight summary
weighted_sample |>
  summarise(
    min_wt = min(base_weight),
    median_wt = median(base_weight),
    mean_wt = mean(base_weight),
    max_wt = max(base_weight),
    cv_wt = sd(base_weight) / mean(base_weight),
    sum_wt = sum(base_weight)
  ) |>
  pivot_longer(everything(), names_to = "Statistic", values_to = "Value") |>
  mutate(Value = format(round(Value, 2), big.mark = ",")) |>
  gt() |>
  tab_header(title = "Base Weight Distribution Summary")
```

## Non-Response Adjustment

### Weighting Class Adjustment

Bagi sampel menjadi **weighting classes** (kelompok homogen terhadap response propensity). Dalam setiap class, inflate weight respondent untuk "mewakili" non-respondent:

$$
w_i^{NR} = w_i \times \frac{n_c}{n_c^{resp}}
$$ {#eq-nr-adj}

di mana $n_c$ = total sampled in class $c$, $n_c^{resp}$ = respondents in class $c$.

### Propensity Model Adjustment

Model probabilitas response menggunakan logistic regression:

$$
\hat{p}_i = P(\text{respond} \mid X_i) = \text{logit}^{-1}(\beta_0 + \beta_1 X_{i1} + \ldots)
$$ {#eq-propensity}

Non-response adjusted weight: $w_i^{NR} = w_i / \hat{p}_i$

::: {.panel-tabset}

### R

```{r}
#| label: nr-adjustment

set.seed(2024)

# Simulate non-response: higher income & urban less likely to respond
weighted_sample <- weighted_sample |>
  mutate(
    resp_prob = plogis(-0.5 + 0.3 * (urban_rural == "Rural") -
                         0.00000005 * income + 0.05 * household_size),
    responded = rbinom(n(), 1, resp_prob)
  )

response_rate <- mean(weighted_sample$responded)

# Method 1: Weighting class adjustment (by urban/rural × region)
nr_adj <- weighted_sample |>
  mutate(wt_class = paste(urban_rural, region)) |>
  mutate(
    n_class = n(),
    n_resp = sum(responded),
    nr_factor = n_class / n_resp,
    .by = wt_class
  ) |>
  mutate(
    wt_nr = ifelse(responded == 1, base_weight * nr_factor, NA)
  )

respondents <- nr_adj |> filter(responded == 1)

# Compare estimates
est_base <- weighted.mean(respondents$income, respondents$base_weight)
est_nr <- weighted.mean(respondents$income, respondents$wt_nr)
true_mean <- mean(pop$income)

tibble(
  Estimate = c("True Population Mean", "Respondents (base weight only)",
               "Respondents (NR adjusted)"),
  Mean_Income = c(true_mean, est_base, est_nr),
  Bias = c(0, est_base - true_mean, est_nr - true_mean)
) |>
  mutate(across(c(Mean_Income, Bias), ~format(round(.), big.mark = ","))) |>
  gt() |>
  tab_header(
    title = "Non-Response Adjustment Effect",
    subtitle = sprintf("Response rate = %.1f%%", response_rate * 100)
  )
```

## Calibration / Raking

### Konsep

Calibration menyesuaikan weights agar distribusi **marginal** sampel match dengan **known population totals** dari sensus atau administrative data [@deville1992].

**Raking** (iterative proportional fitting) menyesuaikan weights secara bergantian pada setiap variabel sampai konvergensi.

### Implementasi

::: {.panel-tabset}

#### R

```{r}
#| label: calibration-demo
#| fig-cap: "Distribusi weight sebelum dan sesudah calibration."
#| fig-width: 9
#| fig-height: 5

# Known population totals
pop_totals_urban <- pop |>
  count(urban_rural) |>
  mutate(Freq = n) |>
  select(urban_rural, Freq)

pop_totals_region <- pop |>
  count(region) |>
  mutate(Freq = n) |>
  select(region, Freq)

# Design with NR-adjusted weights
des_nr <- svydesign(
  id = ~district,
  weights = ~wt_nr,
  data = respondents
)

# Post-stratification on urban/rural
pop_table_urban <- xtabs(~urban_rural, data = pop)
obs_urban <- sort(unique(respondents$urban_rural))
pop_table_urban <- pop_table_urban[names(pop_table_urban) %in% obs_urban]
des_ps <- postStratify(des_nr, ~urban_rural, pop_table_urban, partial = TRUE)

# Raking on urban/rural AND region
pop_table_region <- xtabs(~region, data = pop)
obs_region <- sort(unique(respondents$region))
pop_table_region <- pop_table_region[names(pop_table_region) %in% obs_region]
des_rake <- rake(des_nr,
                  sample.margins = list(~urban_rural, ~region),
                  population.margins = list(pop_table_urban, pop_table_region))

# Compare
est_pre <- coef(svymean(~income, des_nr))
est_ps <- coef(svymean(~income, des_ps))
est_rake <- coef(svymean(~income, des_rake))

tibble(
  Method = c("True Mean", "Pre-calibration (NR adj only)",
             "Post-stratified (urban/rural)", "Raked (urban/rural + region)"),
  Estimate = c(true_mean, est_pre, est_ps, est_rake),
  Bias = c(0, est_pre - true_mean, est_ps - true_mean, est_rake - true_mean)
) |>
  mutate(across(c(Estimate, Bias), ~format(round(.), big.mark = ","))) |>
  gt() |>
  tab_header(title = "Calibration Effect on Estimates")
```

```{python}
#| label: calibration-demo-py

import numpy as np
import pandas as pd

# Simplified raking demonstration in Python
sample_py = r.respondents.copy()
pop_py = r.pop.copy()

# Known population proportions
pop_urban_prop = (pop_py['urban_rural'] == 'Urban').mean()
pop_region_prop = pop_py['region'].value_counts(normalize=True).to_dict()

# Manual single-step calibration (urban/rural)
weights = sample_py['wt_nr'].values.copy()
urban_mask = (sample_py['urban_rural'] == 'Urban').values

# Adjust urban weights
current_urban_prop = weights[urban_mask].sum() / weights.sum()
urban_factor = pop_urban_prop / current_urban_prop
rural_factor = (1 - pop_urban_prop) / (1 - current_urban_prop)

cal_weights = weights.copy()
cal_weights[urban_mask] *= urban_factor
cal_weights[~urban_mask] *= rural_factor

# Compare
pre_mean = np.average(sample_py['income'], weights=weights)
post_mean = np.average(sample_py['income'], weights=cal_weights)
true_mean = pop_py['income'].mean()

results = pd.DataFrame({
    'Method': ['True Mean', 'Pre-calibration', 'Post-calibration (urban/rural)'],
    'Estimate': [f'{true_mean:,.0f}', f'{pre_mean:,.0f}', f'{post_mean:,.0f}'],
    'Bias': [0, f'{pre_mean - true_mean:,.0f}', f'{post_mean - true_mean:,.0f}']
})
results
```

## Weight Trimming

### Mengapa Trimming?

Extreme weights meningkatkan varians estimasi. Trimming meng-cap weights pada percentile tertentu (misalnya P95 atau P99) atau pada kelipatan dari median.

$$
w_i^{trim} = \min(w_i, w_{max})
$$

::: {.callout-warning}
## Trade-off
Trimming **mengurangi varians** tapi **memperkenalkan bias**. Aturan praktis: trim jika $\text{CV}(w) > 1.0$ atau jika ada weights yang >10× median [@potter1990].
:::

### Implementasi

::: {.panel-tabset}

#### R

```{r}
wts <- weights(des_rake)

# Trimming at various percentiles
trim_levels <- c(0.95, 0.97, 0.99, 1.00)

trim_results <- map_dfr(trim_levels, function(p) {
  cap <- quantile(wts, p)
  trimmed <- pmin(wts, cap)
  est <- weighted.mean(respondents$income, trimmed)
  tibble(
    percentile = paste0("P", p * 100),
    cap_value = round(cap),
    mean_wt = mean(trimmed),
    cv_wt = sd(trimmed) / mean(trimmed),
    estimate = est,
    bias = est - true_mean
  )
})

trim_results |>
  mutate(across(where(is.numeric), ~format(round(., 2), big.mark = ","))) |>
  gt() |>
  tab_header(title = "Weight Trimming: Effect on CV and Estimates")
```

## Effective Sample Size Setelah Weighting

$$
n_{eff} = \frac{(\sum w_i)^2}{\sum w_i^2} = \frac{n}{1 + \text{CV}(w)^2}
$$ {#eq-neff-weight}

::: {.panel-tabset}

### R

```{r}
#| label: tbl-neff
#| tbl-cap: "Effective sample size pada setiap tahap weighting."

calc_neff <- function(w) {
  sum(w)^2 / sum(w^2)
}

tibble(
  Stage = c("Base weight", "NR adjusted", "Post-raking", "Trimmed (P95)"),
  n_actual = nrow(respondents),
  n_eff = c(
    calc_neff(respondents$base_weight),
    calc_neff(respondents$wt_nr),
    calc_neff(weights(des_rake)),
    calc_neff(trimmed_wts)
  ),
  CV_weights = c(
    sd(respondents$base_weight) / mean(respondents$base_weight),
    sd(respondents$wt_nr) / mean(respondents$wt_nr),
    sd(weights(des_rake)) / mean(weights(des_rake)),
    sd(trimmed_wts) / mean(trimmed_wts)
  ),
  DEFF_w = c(
    1 + (sd(respondents$base_weight) / mean(respondents$base_weight))^2,
    1 + (sd(respondents$wt_nr) / mean(respondents$wt_nr))^2,
    1 + (sd(weights(des_rake)) / mean(weights(des_rake)))^2,
    1 + (sd(trimmed_wts) / mean(trimmed_wts))^2
  )
) |>
  mutate(across(c(n_eff, CV_weights, DEFF_w), ~round(., 2))) |>
  gt() |>
  tab_header(title = "Effective Sample Size by Weighting Stage")
```

## Weight Diagnostics

### Checklist Diagnostik

1. **Distribusi weight:** histogram, box plot — cari extreme values
2. **CV of weights:** target CV < 1.0; CV > 1.5 problematic
3. **Sum of weights:** harus ≈ $N$ (population size)
4. **Ratio max/min weight:** jika > 20, pertimbangkan trimming
5. **Marginal distributions:** setelah calibration, distribusi variabel kunci harus match populasi

::: {.panel-tabset}

::: {.takeaways}
### Key Takeaways

1. **Base weight = 1/π_i** — ini adalah starting point. Jangan pernah analyze data survei tanpa weight.
2. **Non-response adjustment** mengoreksi bias dari unit yang tidak merespons, menggunakan weighting classes atau propensity models.
3. **Calibration/raking** menyesuaikan weight agar distribusi match dengan populasi yang diketahui — mengurangi bias dan bisa meningkatkan presisi.
4. **Weight trimming** mengurangi varians dari extreme weights, tapi trade-off dengan sedikit bias.
5. **Selalu lakukan diagnostik:** cek CV of weights, distribusi, dan marginal conformity setelah setiap tahap adjustment.
:::

### Exercises

Bandingkan estimasi mean income: (a) tanpa adjustment, (b) dengan weighting class adjustment (urban/rural × education), (c) dengan propensity model. Mana yang paling efektif mengurangi bias?

3. **Calibration exercise.** Lakukan raking pada sampel menggunakan distribusi populasi: urban/rural, region, dan education level. Berapa persen pengurangan bias? Berapa n_eff setelah raking?
