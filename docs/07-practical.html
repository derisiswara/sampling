3 10 Kesalahan Paling Sering

3.1 1. Mengabaikan Design Effect

Penting

Masalah: Menghitung sample size dengan formula SRS, lalu menggunakan cluster sampling di lapangan.

Dampak: Presisi aktual jauh lebih rendah dari yang diharapkan. Jika DEFF = 3, presisi Anda setara dengan n/3.

Solusi: Selalu kalikan n_SRS dengan DEFF. Jika ICC tidak diketahui, gunakan DEFF = 2.0 sebagai default konservatif.

3.2 2. Mengabaikan Non-Response

Masalah: Menargetkan n = 400 tapi response rate hanya 60% → effective n = 240.

Solusi: \(n_{final} = n_{target} / (1 - r)\). Budget untuk follow-up.

3.3 3. Menganalisis Tanpa Weight

Masalah: Menggunakan mean(data$y) alih-alih svymean(~y, design).

Dampak: Biased point estimates dan underestimated standard errors.

3.4 4. Melaporkan Unweighted Counts

Masalah: “Tabel menunjukkan 45% responden tinggal di urban.” Ini unweighted proportion — bukan estimasi populasi.

3.5 5. Subset Data Sebelum svydesign()

Masalah: Memfilter data untuk subgroup analisis sebelum mendefinisikan survey design → SE underestimated.

Solusi: Gunakan subset() pada design object.

3.6 6. Memperlakukan Cluster Effect Sebagai Fixed

Masalah: Menggunakan fixed effects untuk cluster ID alih-alih memperlakukannya sebagai random effect / PSU dalam survey design.

3.7 7. Tidak Memeriksa Kualitas Frame

Masalah: Langsung sampling dari frame tanpa assessment → coverage error tidak terdeteksi.

3.8 8. Over-Stratification

Masalah: Terlalu banyak strata → beberapa strata memiliki n < 10 → estimasi tidak stabil.

Rule of thumb: Minimum 2 PSU per stratum untuk estimasi varians.

3.9 9. Mengabaikan Multi-Objective

Masalah: Menghitung n hanya untuk satu variabel, padahal survei mengukur 20 variabel → beberapa variabel underpowered.

3.10 10. Tidak Melakukan Sensitivity Analysis

Masalah: Mempercayai satu set asumsi (σ, DEFF, response rate) tanpa menguji bagaimana hasil berubah jika asumsi dilanggar.

# FULL SURVEY WORKFLOW TEMPLATE
# ============================================================

# --- 0. Setup ---
library(dplyr)
library(survey)
library(srvyr)
library(ggplot2)

# --- 1. Load Population / Frame ---
pop <- load_population()
N <- nrow(pop)

# --- 2. Design Parameters ---
config <- list(
  n_target      = 300,           # target sample size
  n_psu         = 15,            # number of PSUs (districts)
  m_per_psu     = 20,            # SSU per PSU
  strat_var     = "region",      # stratification variable
  psu_var       = "district",    # PSU identifier
  target_vars   = c("income", "expenditure", "satisfaction"),
  deff_assumed  = 2.0,
  response_rate = 0.75
)

# Adjusted sample size
n_adjusted <- ceiling(config$n_target * config$deff_assumed / config$response_rate)

# --- 3. Sample Selection ---
set.seed(2024)

# Stage 1: Select PSUs (stratified PPS)
district_info <- pop |>
  summarise(N_d = n(), .by = district) |>
  mutate(prob_d = N_d / sum(N_d))

selected_psus <- pop |>
  distinct(district, region) |>
  left_join(district_info, by = "district") |>
  group_by(region) |>
  group_modify(~ slice_sample(.x,
                              n = min(config$n_psu, nrow(.x)),
                              weight_by = N_d)) |>
  ungroup() |>
  pull(district)

# Stage 2: SRS within PSUs
survey_sample <- pop |>
  filter(district %in% selected_psus) |>
  group_by(district) |>
  group_modify(~ slice_sample(.x, n = min(config$m_per_psu, nrow(.x)))) |>
  ungroup()

# --- 4. Compute Weights ---
survey_sample <- survey_sample |>
  left_join(district_info, by = "district") |>
  mutate(
    pi_1 = length(selected_psus) * prob_d,  # PSU selection prob
    pi_2 = config$m_per_psu / N_d,          # SSU selection prob
    base_weight = 1 / (pi_1 * pi_2)
  )

# --- 5. Setup Survey Design ---
des <- svydesign(
  id = ~district,
  strata = ~region,
  weights = ~base_weight,
  data = survey_sample,
  nest = TRUE
)

# --- 6. Calibration (Raking) ---
pop_urban <- xtabs(~urban_rural, data = pop)
pop_region <- xtabs(~region, data = pop)

des_cal <- rake(
  des,
  sample.margins = list(~urban_rural, ~region),
  population.margins = list(pop_urban, pop_region)
)

# --- 7. Analysis ---
# Overall means
overall <- svymean(~income + expenditure, des_cal)

# By subgroup
by_region <- svyby(~income, ~region, des_cal, svymean)
by_urban <- svyby(~income, ~urban_rural, des_cal, svymean)

# Regression
model <- svyglm(income ~ urban_rural + education + household_size,
                 design = des_cal)

# --- 8. Diagnostics ---
final_wts <- weights(des_cal)
wt_diagnostics <- tibble(
  n_actual = nrow(survey_sample),
  n_eff = sum(final_wts)^2 / sum(final_wts^2),
  cv_weights = sd(final_wts) / mean(final_wts),
  deff_w = 1 + (sd(final_wts) / mean(final_wts))^2,
  sum_weights = sum(final_wts),
  ratio_max_min = max(final_wts) / min(final_wts)
)

# --- 9. Results Summary ---
results_summary <- tibble(
  Variable = c("Income", "Expenditure"),
  Estimate = format(round(coef(overall)), big.mark = ","),
  SE = format(round(SE(overall)), big.mark = ","),
  CI_Lower = format(round(confint(overall)[, 1]), big.mark = ","),
  CI_Upper = format(round(confint(overall)[, 2]), big.mark = ",")
)

results_summary |>
  gt() |>
  tab_header(
    title = "Survey Results Summary",
    subtitle = sprintf("n = %d | n_eff = %.0f | CV(w) = %.3f",
                       nrow(survey_sample), wt_diagnostics$n_eff,
                       wt_diagnostics$cv_weights)
  )

import numpy as np
import pandas as pd
from scipy.stats import norm

np.random.seed(2024)

# --- 1. Load Data ---
pop_py = r.pop.copy()
N = len(pop_py)

# --- 2. Two-stage Sampling ---
n_psu = 15
m_per_psu = 20

district_info = pop_py.groupby('district').size().reset_index(name='N_d')
district_info['prob'] = district_info['N_d'] / district_info['N_d'].sum()

# PPS selection
selected = district_info.sample(n=n_psu, weights='prob',
                                 random_state=2024)['district'].values

# SRS within PSU
samples = []
for d in selected:
    d_data = pop_py[pop_py['district'] == d]
    s = d_data.sample(n=min(m_per_psu, len(d_data)), random_state=2024)
    d_info = district_info[district_info['district'] == d].iloc[0]
    s = s.copy()
    s['pi_1'] = n_psu * d_info['prob']
    s['pi_2'] = m_per_psu / d_info['N_d']
    s['weight'] = 1 / (s['pi_1'] * s['pi_2'])
    samples.append(s)

survey_sample = pd.concat(samples)

# --- 3. Weighted Analysis ---
weighted_mean = np.average(survey_sample['income'],
                           weights=survey_sample['weight'])
true_mean = pop_py['income'].mean()

# n_eff
w = survey_sample['weight'].values
n_eff = w.sum()**2 / (w**2).sum()
cv_w = w.std() / w.mean()

# By region
by_region = survey_sample.groupby('region').apply(
    lambda g: pd.Series({
        'weighted_mean': np.average(g['income'], weights=g['weight']),
        'n': len(g)
    })
).reset_index()

results = pd.DataFrame({
    'Metric': ['Weighted Mean Income', 'True Mean', 'Bias',
               'n', 'n_eff', 'CV(weights)'],
    'Value': [f'{weighted_mean:,.0f}', f'{true_mean:,.0f}',
              f'{weighted_mean - true_mean:,.0f}',
              str(len(survey_sample)), f'{n_eff:.1f}', f'{cv_w:.4f}']
})

5 Case Study: Survei Kepuasan Pelanggan Bank

5.1 Konteks

Sebuah bank dengan 50.000 nasabah aktif di 10 kota ingin mengukur kepuasan pelanggan (skor 1–5). Tujuan:

  1. Estimasi overall mean kepuasan dengan MoE ≤ 0.15 (95% CI)
  2. Estimasi per kota (10 kota) dengan MoE ≤ 0.30
  3. Perbandingan kepuasan nasabah digital vs conventional
  4. Identifikasi faktor-faktor yang mempengaruhi kepuasan

# Parameters
N_bank <- 50000
moe_overall <- 0.15
moe_city <- 0.30
alpha <- 0.05
z <- qnorm(1 - alpha/2)
sigma_est <- 1.0      # estimated SD of satisfaction (1-5 scale)
deff <- 1.5            # within-city clustering mild
response_rate <- 0.65  # expected

# Overall sample size
n_overall <- ceiling(z^2 * sigma_est^2 / moe_overall^2)
n_overall_fpc <- ceiling(n_overall / (1 + n_overall / N_bank))

# Per city (10 cities, each needs n_city)
n_per_city <- ceiling(z^2 * sigma_est^2 / moe_city^2)
n_cities_total <- n_per_city * 10

# Take maximum
n_srs <- max(n_overall_fpc, n_cities_total)

# Adjust
n_adjusted <- ceiling(n_srs * deff / response_rate)

  left_join(allocation |> select(stratum, n_h_prop), by = "stratum") |>

## Checklist Sebelum Sampling

::: {.callout-tip}
## Pre-Sampling Checklist

### 1. Sampling Frame
- [ ] Apakah frame tersedia dan up-to-date?
- [ ] Berapa coverage rate terhadap target population?
- [ ] Apakah ada duplikasi, undercoverage, atau overcoverage?
- [ ] Kapan terakhir frame diupdate?

### 2. Stratifikasi
- [ ] Variabel stratifikasi apa yang tersedia di frame?
- [ ] Apakah variabel tersebut berkorelasi dengan variabel survei utama?
- [ ] Berapa strata optimal? (jangan terlalu banyak — minimum n per stratum ≥ 20)

### 3. Budget & Logistik
- [ ] Berapa total budget untuk fieldwork?
- [ ] Berapa cost per interview (termasuk transportasi, honorarium, supervisi)?
- [ ] Berapa banyak interviewer tersedia dan terlatih?
- [ ] Berapa lama periode fieldwork?

### 4. Subgroup Analysis
- [ ] Subgroup mana yang harus dianalisis secara terpisah?
- [ ] Apakah perlu estimasi per subgroup atau hanya overall?
- [ ] Berapa presisi minimum per subgroup?

### 5. Non-Response
- [ ] Berapa expected response rate (dari survei sebelumnya)?
- [ ] Apakah ada strategi follow-up untuk non-respondent?
- [ ] Data apa yang tersedia untuk non-response adjustment?

### 6. Quality Control
- [ ] Bagaimana verifikasi data di lapangan?
- [ ] Berapa persen back-checking yang dilakukan?
- [ ] Apakah ada real-time monitoring dashboard?

## 10 Kesalahan Paling Sering

### 1. Mengabaikan Design Effect

::: {.callout-important}
**Masalah:** Menghitung sample size dengan formula SRS, lalu menggunakan cluster sampling di lapangan.

**Dampak:** Presisi aktual jauh lebih rendah dari yang diharapkan. Jika DEFF = 3, presisi Anda setara dengan n/3.

**Solusi:** Selalu kalikan n_SRS dengan DEFF. Jika ICC tidak diketahui, gunakan DEFF = 2.0 sebagai default konservatif.
:::

### 2. Mengabaikan Non-Response

**Masalah:** Menargetkan n = 400 tapi response rate hanya 60% → effective n = 240.

**Solusi:** $n_{final} = n_{target} / (1 - r)$. Budget untuk follow-up.

### 3. Menganalisis Tanpa Weight

**Masalah:** Menggunakan `mean(data$y)` alih-alih `svymean(~y, design)`.

**Dampak:** Biased point estimates dan underestimated standard errors.

### 4. Melaporkan Unweighted Counts

**Masalah:** "Tabel menunjukkan 45% responden tinggal di urban." Ini unweighted proportion — bukan estimasi populasi.

### 5. Subset Data Sebelum svydesign()

**Masalah:** Memfilter data untuk subgroup analisis sebelum mendefinisikan survey design → SE underestimated.

**Solusi:** Gunakan `subset()` pada design object.

### 6. Memperlakukan Cluster Effect Sebagai Fixed

**Masalah:** Menggunakan fixed effects untuk cluster ID alih-alih memperlakukannya sebagai random effect / PSU dalam survey design.

### 7. Tidak Memeriksa Kualitas Frame

**Masalah:** Langsung sampling dari frame tanpa assessment → coverage error tidak terdeteksi.

### 8. Over-Stratification

**Masalah:** Terlalu banyak strata → beberapa strata memiliki n < 10 → estimasi tidak stabil.

**Rule of thumb:** Minimum 2 PSU per stratum untuk estimasi varians.

### 9. Mengabaikan Multi-Objective

**Masalah:** Menghitung n hanya untuk satu variabel, padahal survei mengukur 20 variabel → beberapa variabel underpowered.

### 10. Tidak Melakukan Sensitivity Analysis

**Masalah:** Mempercayai satu set asumsi (σ, DEFF, response rate) tanpa menguji bagaimana hasil berubah jika asumsi dilanggar.

```{r}
#| label: full-workflow
#| code-fold: false

# ============================================================
# FULL SURVEY WORKFLOW TEMPLATE
# ============================================================

# --- 0. Setup ---
library(dplyr)
library(survey)
library(srvyr)
library(ggplot2)

# --- 1. Load Population / Frame ---
pop <- load_population()
N <- nrow(pop)

# --- 2. Design Parameters ---
config <- list(
  n_target      = 300,           # target sample size
  n_psu         = 15,            # number of PSUs (districts)
  m_per_psu     = 20,            # SSU per PSU
  strat_var     = "region",      # stratification variable
  psu_var       = "district",    # PSU identifier
  target_vars   = c("income", "expenditure", "satisfaction"),
  deff_assumed  = 2.0,
  response_rate = 0.75
)

# Adjusted sample size
n_adjusted <- ceiling(config$n_target * config$deff_assumed / config$response_rate)

# --- 3. Sample Selection ---
set.seed(2024)

# Stage 1: Select PSUs (stratified PPS)
district_info <- pop |>
  summarise(N_d = n(), .by = district) |>
  mutate(prob_d = N_d / sum(N_d))

selected_psus <- pop |>
  distinct(district, region) |>
  left_join(district_info, by = "district") |>
  group_by(region) |>
  group_modify(~ slice_sample(.x,
                              n = min(config$n_psu, nrow(.x)),
                              weight_by = N_d)) |>
  ungroup() |>
  pull(district)

# Stage 2: SRS within PSUs
survey_sample <- pop |>
  filter(district %in% selected_psus) |>
  group_by(district) |>
  group_modify(~ slice_sample(.x, n = min(config$m_per_psu, nrow(.x)))) |>
  ungroup()

# --- 4. Compute Weights ---
survey_sample <- survey_sample |>
  left_join(district_info, by = "district") |>
  mutate(
    pi_1 = length(selected_psus) * prob_d,  # PSU selection prob
    pi_2 = config$m_per_psu / N_d,          # SSU selection prob
    base_weight = 1 / (pi_1 * pi_2)
  )

# --- 5. Setup Survey Design ---
des <- svydesign(
  id = ~district,
  strata = ~region,
  weights = ~base_weight,
  data = survey_sample,
  nest = TRUE
)

# --- 6. Calibration (Raking) ---
pop_urban <- xtabs(~urban_rural, data = pop)
pop_region <- xtabs(~region, data = pop)

des_cal <- rake(
  des,
  sample.margins = list(~urban_rural, ~region),
  population.margins = list(pop_urban, pop_region)
)

# --- 7. Analysis ---
# Overall means
overall <- svymean(~income + expenditure, des_cal)

# By subgroup
by_region <- svyby(~income, ~region, des_cal, svymean)
by_urban <- svyby(~income, ~urban_rural, des_cal, svymean)

# Regression
model <- svyglm(income ~ urban_rural + education + household_size,
                 design = des_cal)

# --- 8. Diagnostics ---
final_wts <- weights(des_cal)
wt_diagnostics <- tibble(
  n_actual = nrow(survey_sample),
  n_eff = sum(final_wts)^2 / sum(final_wts^2),
  cv_weights = sd(final_wts) / mean(final_wts),
  deff_w = 1 + (sd(final_wts) / mean(final_wts))^2,
  sum_weights = sum(final_wts),
  ratio_max_min = max(final_wts) / min(final_wts)
)

# --- 9. Results Summary ---
results_summary <- tibble(
  Variable = c("Income", "Expenditure"),
  Estimate = format(round(coef(overall)), big.mark = ","),
  SE = format(round(SE(overall)), big.mark = ","),
  CI_Lower = format(round(confint(overall)[, 1]), big.mark = ","),
  CI_Upper = format(round(confint(overall)[, 2]), big.mark = ",")
)

results_summary |>
  gt() |>
  tab_header(
    title = "Survey Results Summary",
    subtitle = sprintf("n = %d | n_eff = %.0f | CV(w) = %.3f",
                       nrow(survey_sample), wt_diagnostics$n_eff,
                       wt_diagnostics$cv_weights)
  )
```

# ============================================================
# FULL SURVEY WORKFLOW TEMPLATE (Python)
# ============================================================

import numpy as np
import pandas as pd
from scipy.stats import norm

np.random.seed(2024)

# --- 1. Load Data ---
pop_py = r.pop.copy()
N = len(pop_py)

# --- 2. Two-stage Sampling ---
n_psu = 15
m_per_psu = 20

district_info = pop_py.groupby('district').size().reset_index(name='N_d')
district_info['prob'] = district_info['N_d'] / district_info['N_d'].sum()

# PPS selection
selected = district_info.sample(n=n_psu, weights='prob',
                                 random_state=2024)['district'].values

# SRS within PSU
samples = []
for d in selected:
    d_data = pop_py[pop_py['district'] == d]
    s = d_data.sample(n=min(m_per_psu, len(d_data)), random_state=2024)
    d_info = district_info[district_info['district'] == d].iloc[0]
    s = s.copy()
    s['pi_1'] = n_psu * d_info['prob']
    s['pi_2'] = m_per_psu / d_info['N_d']
    s['weight'] = 1 / (s['pi_1'] * s['pi_2'])
    samples.append(s)

survey_sample = pd.concat(samples)

# --- 3. Weighted Analysis ---
weighted_mean = np.average(survey_sample['income'],
                           weights=survey_sample['weight'])
true_mean = pop_py['income'].mean()

# n_eff
w = survey_sample['weight'].values
n_eff = w.sum()**2 / (w**2).sum()
cv_w = w.std() / w.mean()

# By region
by_region = survey_sample.groupby('region').apply(
    lambda g: pd.Series({
        'weighted_mean': np.average(g['income'], weights=g['weight']),
        'n': len(g)
    })
).reset_index()

results = pd.DataFrame({
    'Metric': ['Weighted Mean Income', 'True Mean', 'Bias',
               'n', 'n_eff', 'CV(weights)'],
    'Value': [f'{weighted_mean:,.0f}', f'{true_mean:,.0f}',
              f'{weighted_mean - true_mean:,.0f}',
              str(len(survey_sample)), f'{n_eff:.1f}', f'{cv_w:.4f}']
})
results

### Summary: Langkah-langkah Utama Case Study

| Step | Action | Output |
|------|--------|--------|
| 1 | Define objectives & precision requirements | MoE per tujuan |
| 2 | Evaluate frame (database nasabah) | Coverage assessment |
| 3 | Choose design (stratified by city × type) | Design specification |
| 4 | Calculate sample size (multi-objective + DEFF + NR) | n = `r n_adjusted` |
| 5 | Allocate to strata (proportional, min 15/stratum) | Allocation table |
| 6 | Select sample & conduct survey | Raw data |
| 7 | Compute base weights (1/π_i) | Weighted dataset |
| 8 | Adjust for non-response (weighting class) | NR-adjusted weights |
| 9 | Calibrate to known totals (city distribution) | Final weights |
| 10 | Analyze with survey design (svydesign + svymean etc.) | Estimates + SE + CI |
| 11 | Diagnostics (weight distribution, marginals) | QC report |
| 12 | Report (estimates, methodology, limitations) | Final deliverable |

::: {.takeaways}
### Key Takeaways

1. **Gunakan decision flowchart** untuk memilih metode — jangan default ke convenience sampling hanya karena mudah.
2. **Checklist sebelum sampling** mencegah kesalahan yang mahal — terutama frame quality, DEFF, dan non-response rate.
3. **10 kesalahan terbanyak** semuanya bisa dihindari dengan perencanaan yang matang dan pemahaman design-based inference.
4. **Full workflow template** menyediakan kerangka kerja yang reproducible — dari sampling frame hingga estimasi akhir.
5. **Case study menunjukkan** bahwa setiap langkah mempengaruhi kualitas hasil — short-cut di satu langkah memperbesar error di hilir.
