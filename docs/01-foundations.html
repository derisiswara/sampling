Mengapa sampling, framework konseptual, dan konsep-konsep kunci

1 Mengapa Sampling?

Sampling bukan sekadar “mengambil sebagian data” — ia adalah framework inferensial yang memungkinkan kita membuat pernyataan tentang populasi berdasarkan subset yang jauh lebih kecil. Tiga argumen utama:

  1. Biaya & waktu. Sensus Penduduk Indonesia (SP2020) membutuhkan anggaran triliunan rupiah dan ratusan ribu petugas. Survei sampel seperti SUSENAS mencakup ~300.000 rumah tangga dari ~70 juta — cukup untuk estimasi hingga level kabupaten.

  2. Akurasi. Paradoks: sampel yang dirancang baik sering lebih akurat daripada sensus. Sensus rentan terhadap non-sampling error (pelatihan petugas yang tidak merata, kelelahan, data entry error) yang terakumulasi pada skala besar (Cochran, 1977).

  3. Feasibility. Beberapa pengukuran bersifat destruktif (uji kualitas produk) atau populasi tidak terjangkau seluruhnya (hidden populations).

CatatanPrinsip Fundamental

Kualitas sampel ditentukan oleh desain, bukan hanya ukuran. Sampel 1.000 yang dirancang baik mengalahkan sampel 100.000 yang convenience-based — sebagaimana ditunjukkan oleh kegagalan Literary Digest poll 1936 (n = 2.4 juta) vs. keberhasilan Gallup poll (n = 50.000).

2.2 Definisi

  • Target population: seluruh unit yang menjadi subjek inferensi. Contoh: seluruh rumah tangga di Indonesia.
  • Study population: subset target population yang secara praktis dapat dijangkau. Contoh: rumah tangga yang tinggal di wilayah yang aman dan accessible.
  • Sampling frame: daftar operasional dari mana sampel ditarik. Contoh: daftar blok sensus SP2020 (BPS), daftar nasabah aktif bank, database nomor telepon.

2.3 Coverage Error

Coverage error = perbedaan antara target population dan sampling frame:

  • Undercoverage: unit di populasi tidak ada di frame. Contoh: rumah tangga baru yang belum tercatat di SP2020; pelanggan baru yang belum masuk database.
  • Overcoverage: unit di frame bukan bagian dari populasi. Contoh: rumah tangga yang sudah pindah/meninggal tapi masih tercatat.
  • Duplikasi: unit muncul >1 kali di frame.
PeringatanPitfall

Coverage error adalah salah satu sumber bias paling berbahaya karena tidak bisa dideteksi dari data sampel — kita tidak tahu apa yang tidak kita ketahui. Selalu evaluasi kualitas frame sebelum mendesain survei.

3 Konsep Kunci

3.1 Sampling Error vs. Non-Sampling Error

Sampling error menurun dengan bertambahnya \(n\). Non-sampling error bisa meningkat seiring bertambahnya \(n\) jika quality control memburuk.

Sampling error adalah variabilitas estimasi yang timbul karena kita hanya mengobservasi sebagian populasi. Jika kita mengambil sampel berbeda, kita mendapat estimasi berbeda — variabilitas inilah sampling error.

Non-sampling error mencakup semua error lainnya:

  • Measurement error: responden salah paham pertanyaan
  • Processing error: salah input data
  • Non-response error: responden menolak/tidak bisa dihubungi
  • Coverage error: frame tidak mencakup populasi

3.2 Bias vs. Variance Trade-off

Untuk estimator \(\hat{\theta}\) dari parameter \(\theta\):

\[ \text{MSE}(\hat{\theta}) = \text{Bias}^2(\hat{\theta}) + \text{Var}(\hat{\theta}) \tag{1}\]

di mana:

  • \(\text{Bias}(\hat{\theta}) = E[\hat{\theta}] - \theta\)
  • \(\text{Var}(\hat{\theta}) = E[(\hat{\theta} - E[\hat{\theta}])^2]\)

Estimator yang unbiased memiliki \(E[\hat{\theta}] = \theta\), sehingga \(\text{MSE} = \text{Var}\). Namun dalam praktik, sedikit bias kadang bisa diterima jika menghasilkan penurunan varians yang substansial.

3.3 Parameter, Estimator, Estimate

Konsep Definisi Contoh
Parameter (\(\theta\)) Nilai tetap (unknown) di populasi \(\bar{Y} = \frac{1}{N}\sum_{i=1}^N Y_i\)
Estimator (\(\hat{\theta}\)) Fungsi dari data sampel (random variable) \(\bar{y} = \frac{1}{n}\sum_{i=1}^n y_i\)
Estimate Nilai numerik spesifik dari estimator \(\bar{y} = 4{,}250{,}000\)

3.4 Margin of Error dan Confidence Interval

Margin of Error (MoE) pada level kepercayaan \((1-\alpha)\):

\[ \text{MoE} = z_{1-\alpha/2} \times \text{SE}(\hat{\theta}) \tag{2}\]

Confidence Interval:

\[ \text{CI}_{1-\alpha} = \hat{\theta} \pm \text{MoE} \tag{3}\]

TipInterpretasi yang Benar

CI 95% bukan berarti “ada probabilitas 95% parameter ada di interval ini.” Interpretasi yang benar: jika kita mengulangi prosedur sampling berkali-kali, 95% dari interval yang dihasilkan akan mengandung parameter sebenarnya.

import numpy as np
import pandas as pd

import matplotlib.ticker as mticker

    ax.set_xlabel('Sample Mean (Income)')
    ax.set_ylabel('Density')
    ax.xaxis.set_major_formatter(mticker.FuncFormatter(
        lambda x, p: f'{x:,.0f}'))
    ax.legend(fontsize=8)

fig.suptitle('Central Limit Theorem: Distribusi Sampling Mean',
             fontweight='bold', fontsize=13)
plt.tight_layout()
plt.show()

6 Sampling Error vs. Sample Size

Hubungan antara sampling error dan ukuran sampel bersifat diminishing returns — peningkatan presisi melambat seiring bertambahnya \(n\).

  • R
  • Python
Kode 
set.seed(2024)
n_values <- c(10, 20, 30, 50, 100, 200, 300, 500, 750, 1000)
B <- 2000

se_sim <- map_dfr(n_values, function(n) {
  means <- replicate(B, mean(sample(pop$income, n)))
  tibble(n = n, se_empirical = sd(means))
})

pop_sd <- sd(pop$income)
se_sim <- se_sim |>
  mutate(se_theoretical = pop_sd / sqrt(n))

ggplot(se_sim, aes(x = n)) +
  geom_point(aes(y = se_empirical, color = "Empirical (simulasi)"),
             size = 3) +
  geom_line(aes(y = se_theoretical, color = "Theoretical (σ/√n)"),
            linewidth = 1) +
  scale_color_manual(values = c("Empirical (simulasi)" = "#2563eb",
                                "Theoretical (σ/√n)" = "#dc2626")) +
  scale_x_continuous(breaks = n_values) +
  scale_y_continuous(labels = label_comma()) +
  labs(
    title = "Standard Error vs. Sample Size",
    subtitle = "SE menurun proporsional dengan 1/√n — diminishing returns",
    x = "Sample Size (n)",
    y = "Standard Error of Mean",
    color = NULL
  )
Kode 
import numpy as np
import matplotlib.pyplot as plt
import matplotlib.ticker as mticker

np.random.seed(2024)

income = r.pop['income'].values
pop_sd = income.std(ddof=0)

n_values = [10, 20, 30, 50, 100, 200, 300, 500, 750, 1000]
B = 2000

se_empirical = []
for n in n_values:
    means = [np.mean(np.random.choice(income, size=n, replace=False))
             for _ in range(B)]
    se_empirical.append(np.std(means, ddof=1))

se_theoretical = [pop_sd / np.sqrt(n) for n in n_values]

fig, ax = plt.subplots(figsize=(8, 5))
ax.scatter(n_values, se_empirical, color='#2563eb', s=60, zorder=5,
           label='Empirical (simulasi)')
ax.plot(n_values, se_theoretical, color='#dc2626', linewidth=1.5,
        label='Theoretical (σ/√n)')
ax.set_xlabel('Sample Size (n)')
ax.set_ylabel('Standard Error of Mean')
ax.set_title('Standard Error vs. Sample Size', fontweight='bold')
ax.legend()
ax.yaxis.set_major_formatter(mticker.FuncFormatter(
    lambda x, p: f'{x:,.0f}'))
plt.tight_layout()
plt.show()
CatatanImplikasi Praktis

Menggandakan presisi (mengurangi SE menjadi setengah) membutuhkan empat kali lipat ukuran sampel (\(n \propto 1/\text{SE}^2\)). Ini adalah alasan utama mengapa penentuan ukuran sampel harus berbasis perhitungan — bukan sekadar “ambil sebanyak mungkin.”

6.1 Key Takeaways

  1. Kualitas sampling ditentukan oleh desain, bukan hanya ukuran. Sampel kecil yang well-designed mengalahkan sampel besar yang biased.
  2. Sampling error hanyalah satu komponen dari Total Survey Error. Non-sampling error (coverage, non-response, measurement) sering lebih besar dan lebih sulit dikendalikan.
  3. Sampling frame adalah titik kritis — gap antara frame dan target population menghasilkan coverage error yang tidak terdeteksi dari data.
  4. Standard error menurun proporsional dengan \(1/\sqrt{n}\) — ada diminishing returns. Perhitungan formal sample size diperlukan untuk menentukan titik optimal.
  5. CLT menjamin distribusi sampling mean mendekati normal untuk \(n\) yang cukup besar, memungkinkan konstruksi CI dan hypothesis testing.

6.2 Exercises

  1. Coverage error analysis. Anda diminta mendesain survei kepuasan pelanggan e-commerce. Frame yang tersedia: database email pelanggan yang pernah bertransaksi dalam 12 bulan terakhir. Identifikasi potensi undercoverage dan overcoverage. Bagaimana dampaknya terhadap estimasi, dan apa mitigasinya?

  2. Bias-variance simulation. Modifikasi simulasi CLT di atas: alih-alih SRS, gunakan sampling yang hanya mengambil rumah tangga urban. Hitung bias dan MSE dari estimator mean income, dan bandingkan dengan SRS. Apa pelajarannya?

  3. TSE decomposition. Sebuah survei telepon memiliki response rate 40%. Diskusikan komponen TSE mana saja yang kemungkinan besar berkontribusi pada total error, dan urutkan dari yang paling berpengaruh.

## Sampling Error vs. Sample Size

Hubungan antara sampling error dan ukuran sampel bersifat **diminishing returns** — peningkatan presisi melambat seiring bertambahnya $n$.

::: {.panel-tabset}

### R

```{r}
#| label: fig-se-vs-n
#| fig-cap: "Standard Error menurun proporsional dengan $1/\\sqrt{n}$. Perhatikan diminishing returns: dari n=30 ke n=100 jauh lebih impactful daripada dari n=500 ke n=1000."
#| fig-width: 8
#| fig-height: 5

set.seed(2024)
n_values <- c(10, 20, 30, 50, 100, 200, 300, 500, 750, 1000)
B <- 2000

se_sim <- map_dfr(n_values, function(n) {
  means <- replicate(B, mean(sample(pop$income, n)))
  tibble(n = n, se_empirical = sd(means))
})

pop_sd <- sd(pop$income)
se_sim <- se_sim |>
  mutate(se_theoretical = pop_sd / sqrt(n))

ggplot(se_sim, aes(x = n)) +
  geom_point(aes(y = se_empirical, color = "Empirical (simulasi)"),
```

### Python

```{python}
#| label: fig-se-vs-n-py
#| fig-cap: "Standard Error vs. Sample Size (Python implementation)."
#| fig-width: 8
#| fig-height: 5

import numpy as np
import matplotlib.pyplot as plt
import matplotlib.ticker as mticker

np.random.seed(2024)

income = r.pop['income'].values
pop_sd = income.std(ddof=0)

n_values = [10, 20, 30, 50, 100, 200, 300, 500, 750, 1000]
B = 2000

se_empirical = []
for n in n_values:
    means = [np.mean(np.random.choice(income, size=n, replace=False))
             for _ in range(B)]
    se_empirical.append(np.std(means, ddof=1))

se_theoretical = [pop_sd / np.sqrt(n) for n in n_values]

fig, ax = plt.subplots(figsize=(8, 5))
ax.scatter(n_values, se_empirical, color='#2563eb', s=60, zorder=5,
           label='Empirical (simulasi)')
ax.plot(n_values, se_theoretical, color='#dc2626', linewidth=1.5,
        label='Theoretical (σ/√n)')
ax.set_xlabel('Sample Size (n)')
ax.set_ylabel('Standard Error of Mean')
ax.set_title('Standard Error vs. Sample Size', fontweight='bold')
ax.legend()
ax.yaxis.set_major_formatter(mticker.FuncFormatter(
    lambda x, p: f'{x:,.0f}'))
plt.tight_layout()
plt.show()
```

::: {.callout-note}
## Implikasi Praktis
Menggandakan presisi (mengurangi SE menjadi setengah) membutuhkan **empat kali lipat** ukuran sampel ($n \propto 1/\text{SE}^2$). Ini adalah alasan utama mengapa penentuan ukuran sampel harus berbasis perhitungan — bukan sekadar "ambil sebanyak mungkin."
:::

::: {.takeaways}
### Key Takeaways

1. **Kualitas sampling ditentukan oleh desain, bukan hanya ukuran.** Sampel kecil yang well-designed mengalahkan sampel besar yang biased.
2. **Sampling error hanyalah satu komponen** dari Total Survey Error. Non-sampling error (coverage, non-response, measurement) sering lebih besar dan lebih sulit dikendalikan.
3. **Sampling frame adalah titik kritis** — gap antara frame dan target population menghasilkan coverage error yang tidak terdeteksi dari data.
4. **Standard error menurun proporsional dengan $1/\sqrt{n}$** — ada diminishing returns. Perhitungan formal sample size diperlukan untuk menentukan titik optimal.
5. **CLT menjamin** distribusi sampling mean mendekati normal untuk $n$ yang cukup besar, memungkinkan konstruksi CI dan hypothesis testing.
